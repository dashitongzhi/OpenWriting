## 维护规则

- 每次新增、删除或重命名项目文件时，都要同步更新本文件。
- 每次修改现有文件时，如果它的职责、位置或 Git 策略发生变化，也要同步更新本文件。
- `Git` 列说明：
  - `YES`：建议纳入 Git 版本管理。
  - `NO`：不应纳入 Git，通常是生成物、缓存或仓库内部数据。
  - `OPTIONAL`：按团队约定决定，当前默认不新增此类条目。
